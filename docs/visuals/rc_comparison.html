RC Calculation: Layout View Properties vs Routing Review

Layout View → Properties (Trusted)

  • Upload → _rebuild_engine_from_nets()
  • engine.calculate_net_rc(net) → core.calculate_net_rc() (per-layer r_per_sq, c_per_um)
  • total_resistance, total_capacitance
  • tau_rc = ohm_ff_to_ps(R, C, "lumped")
  • Displayed: resistance, capacitance, tau_rc, tpd_50 (with driver/load)
→

Routing Review (Current)

  • Run Review → _run_routing_review()
  • compute_for_net(..., rc_model=None)
  • → compute_net_metrics_with_tau() → calculate_net_rc() (same)
  • if not rc_model: tau_ps = ohm_ff_to_ps(totals, lumped)
  • Displayed: r_total, c_total, effective_tau_ps
Potential Differences:
• tau_rc vs effective_tau_ps (same formula now?)
• tpd in properties has extra driver/load modeling — review does not
• If custom_rc_model active in routing_state → overrides R/C/τ in review only
• Vias handling or polygon splitting in split_metal_via_polygons()
• Different layer_info fallback or tech_layers source

This visual highlights where R/C/τ may diverge even if core calculate_net_rc is shared.
